# textual-dataviz

会話中のデータを最適な形式で可視化する汎用スキル。データ種別を自動判定し、ASCII文字・絵文字を用いた視覚表現を生成する。

## When to use

- 「可視化して」「グラフにして」「図にして」「チャートで見せて」
- 「visualize」「show as chart」「draw diagram」
- 「トレンドを見せて」「比較したい」「分布を表示」
- データや数値が会話に含まれており、視覚的理解が有用な場合

## When NOT to use

- データが存在しない場合（推測で図を描かない）
- 複雑な図形・UML図・アーキテクチャ図（別ツール推奨）
- 画像生成が必要な場合（PNG/SVG生成は対象外）
- 精密な数値グラフが必要な場合（matplotlib/Plotly等の画像生成ツール推奨）
- 50行以上の大規模データ（TOP 5-10に絞るか、別ツールを推奨）
- Mermaid/PlantUML等の構造化ダイアグラムが必要な場合

## 実行フロー

### 1. コンテキスト判定

直前の会話・ファイル・データから「何を可視化すべきか」を推定する。

1. 明示的な対象指定を確認する → 指定があれば従う
   - 例: 「このディレクトリ構造を可視化して」→ ツリー表示
2. 会話に数値データが含まれるか確認する
   - 比率・割合 → 横棒グラフ
   - 時系列データ → トレンドチャート
   - カテゴリ別集計 → 棒グラフ + 表
3. データ種別が不明な場合のみ、1回だけユーザーに確認する

---

### 2. データ種別 → 可視化手法マッピング

| データ種別 | 可視化手法 | 適用例 | 出力例 |
|-----------|-----------|-------|-------|
| 比率・割合 | 横棒グラフ | カテゴリ別支出、タスク進捗 | `食費 █████░░░ 65%` |
| 時系列推移 | トレンドチャート | 週次・月次データ推移 | `Week 1: ████░░ 40% ↗️` |
| 完了/未完了 | ヒートマップ | 習慣トラッキング、カレンダー | `✅✅⬜✅✅⬜✅` |
| 構造・階層 | ツリー | ディレクトリ、組織図 | `├── dir1/` |
| 比較（2者以上） | 並列テーブル | 前月比、複数プロジェクト比較 | Markdown表 + 差分色 |
| 分布・集計 | 棒グラフ+表 | 頻度分析、ランキング | `████ 45件` |
| 状態・段階 | ステータス | 段階的進捗、評価 | `🟥🟧🟨🟩 (0-100%)` |
| 依存・フロー | 矢印図 | ワークフロー、依存関係 | `A → B → C` |

**判定ロジック**:
```
1. データに時刻・日付フィールドがある → 時系列推移
2. データに階層構造がある → ツリー
3. データが完了/未完了の2値 → ヒートマップ
4. データに比率・割合が含まれる → 横棒グラフ
5. データが比較対象を含む → 並列テーブル
6. データに評価・段階がある → ステータス
7. データに接続・依存関係がある → 矢印図
8. いずれにも該当しない → 棒グラフ+表
```

---

### 3. 可視化手法の出力ルール

#### 3-1. 横棒グラフ（比率・割合）

バー: `█`(filled) / `░`(empty)、幅20文字（調整可）、ラベル+バー+数値+割合。

```
カテゴリ名  ████████████░░░░░░░░  65% (¥45,000)
```

→ 詳細: `references/patterns.md`「横棒グラフテンプレート」

#### 3-2. トレンドチャート（時系列推移）

週次/月次/日次の推移、トレンド矢印: ↗️(上昇) / →(横ばい) / ↘️(下降)。

```
Week N: ████████████░░░░░░░░ XX% (+YY%)
```

→ 詳細: `references/patterns.md`「トレンドチャートテンプレート」

#### 3-3. ヒートマップ（完了/未完了）

絵文字: ✅(完了) / ⬜(未完了) / ⚫(未記録)、週次カレンダー形式、凡例必須。

```
Week N: ✅✅✅✅✅⬜⬜
```

→ 詳細: `references/patterns.md`「ヒートマップテンプレート」

#### 3-4. ツリー（構造・階層）

記号: `├──`(中間) / `└──`(最後) / `│`(継続)、深さ制限デフォルト3階層。

```
root/
├── dir1/
│   └── file1.txt
└── dir2/
```

→ 詳細: `references/patterns.md`「ツリーテンプレート」

#### 3-5. 並列テーブル（比較）

Markdown表形式、差分絵文字: 🟢(増加) / 🔴(減少) / ⚪(変化なし)。

```
| 項目 | 今月 | 前月 | 前月比 |
|------|------|------|--------|
| 収入 | ¥XXX | ¥XXX | +X% 🟢 |
```

→ 詳細: `references/patterns.md`「比較テーブルテンプレート」

#### 3-6. 棒グラフ+表（分布・集計）

ソート済み（降順）、合計行追加、バー+数値+割合。

```
Alice  ████████████████████ 120件 (40%)
```

→ 詳細: `references/patterns.md`「集計テンプレート」

#### 3-7. ステータス（状態・段階）

段階色: 🟥(0-25%) / 🟧(26-50%) / 🟨(51-75%) / 🟩(76-100%)、凡例必須。

```
設計  🟩🟩🟩🟩 100% (完了)
```

→ 詳細: `references/patterns.md`「ステータステンプレート」

#### 3-8. 矢印図（依存・フロー）

矢印: `→`(順次) / `⇒`(強調) / `↓`(分岐)。複雑なフローは別ツール推奨。

```
A → B → C
```

→ 詳細: `references/patterns.md`「フローテンプレート」

---

### 4. 出力先

| 出力先 | 動作 |
|-------|------|
| ターミナル（デフォルト） | チャットに直接表示する |
| ファイル | 指定パスに書き出す（`output=file:path/to/file.md`） |
| Daily Note | 今日のDaily Noteの末尾に追記する（`output=daily-note`） |

---

### 5. 出力検証

1. データの欠損・推測がないことを確認する
2. 凡例が必要なチャートタイプ（ヒートマップ、ステータス）で凡例が含まれることを確認する
3. バー幅がwidth設定と一致することを確認する
4. ソートが指定通り適用されていることを確認する

---

## エラーハンドリング

| パターン | 検出方法 | 対処 |
|---------|---------|------|
| データなし | コンテキスト確認 | 「可視化対象が見つかりません。データを提示してください」 |
| データ種別不明 | 判定ロジック不一致 | ユーザーに確認（1回のみ） |
| 出力先ファイルなし | ファイル存在チェック | ファイル作成 or エラー表示 |
| Daily Noteなし | 今日のDaily Note確認 | 「Daily Noteが存在しません」 |

## 絶対厳守ルール

### データ取得

- DO: 存在するデータのみを使用する
- DO: 欠損データは明示的に報告する
- DO: 判定ロジックの基準に従ってデータ種別を判定する
- DO NOT: 推測でデータを補完する
- DO NOT: 存在しないデータを生成する

### 可視化手法

- DO: 詳細が必要な場合は `references/patterns.md` を参照する
- DO: バー幅は20文字を基本とする（調整可）
- DO: ヒートマップ・ステータスには凡例を必ず表示する
- DO NOT: 複雑なデータをそのまま表示する（階層は3階層まで、TOP5まで等に簡略化）

### 出力

- DO: デフォルトはターミナル表示とする
- DO: ファイル書き込み前に存在確認する
- DO: Daily Note追記前に存在確認する
- DO NOT: 確認なしにファイルを上書きする

## Examples

### Example 1: Typical（棒グラフ）

**シナリオ**: カテゴリ別支出データがある状態で「可視化して」

**実行**: 「先月の支出をカテゴリ別に可視化して」

**期待動作**:
1. コンテキストから支出データを抽出
2. データ種別を「比率・割合」と判定
3. 横棒グラフを生成、ターミナルに表示

**成功基準**: 横棒グラフが表示され、ラベル+バー+割合が正しく、降順ソート済み

---

### Example 2: Typical（トレンドチャート）

**シナリオ**: 週次データがある状態で「推移を見せて」

**実行**: 「Exercise習慣の週次推移を可視化して」

**期待動作**:
1. コンテキストから週次データを抽出
2. データ種別を「時系列推移」と判定
3. トレンドチャートを生成、トレンド矢印を追加

**成功基準**: 週次推移が表示され、トレンド矢印が正しく、変化率が計算される

---

### Example 3: Boundary（判定不明）

**シナリオ**: データ種別が不明確なデータに対して「可視化して」

**実行**: 「このデータを可視化して」（種別判定できないデータ）

**期待動作**:
1. データ種別判定 → 該当なし
2. ユーザーに確認: 「何を可視化しますか？ (例: 比率、推移、構造)」
3. ユーザー指定に従って可視化

**成功基準**: 適切な確認メッセージが表示され、ユーザー指定に従って可視化実行

## Evals

### Eval 1: Typical — カテゴリ別支出の横棒グラフ

**入力**:
```
会話コンテキストに以下のデータが含まれる:
食費: ¥45,000 (65%), 住居: ¥80,000 (90%), 交通: ¥15,000 (20%)
ユーザー: 「可視化して」
```

**期待出力**:
- 横棒グラフ形式（`█`/`░` バー）
- 降順ソート（住居 → 食費 → 交通）
- 各行にラベル・バー・割合・金額
- バー幅20文字

**成功判定**: バー形式・ソート順・数値すべてが正確

### Eval 2: Boundary — 50行超データの処理

**入力**:
```
会話コンテキストに60行の売上データが含まれる
ユーザー: 「可視化して」
```

**期待出力**:
- TOP 5-10に絞った棒グラフを出力
- または「データが大量です。TOP Nに絞りますか？」と確認

**成功判定**: 大量データをそのまま表示せず、適切に絞り込みまたは確認を行う

### Eval 3: Failure — データなしでの可視化要求

**入力**:
```
会話コンテキストに数値データなし
ユーザー: 「可視化して」
```

**期待出力**:
- エラーメッセージ: 「可視化対象が見つかりません。データを提示してください」
- 推測でデータを生成しない

**成功判定**: 適切なエラーメッセージが表示され、架空データが生成されない
