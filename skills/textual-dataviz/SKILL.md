# textual-dataviz

会話中のデータを最適な形式で可視化する汎用スキル。データ種別を自動判定し、ASCII文字・絵文字を用いた視覚表現を生成します。

## When to use

- 「可視化して」「グラフにして」「図にして」「チャートで見せて」
- 「visualize」「show as chart」「draw diagram」
- 「トレンドを見せて」「比較したい」「分布を表示」
- データや数値が会話に含まれており、視覚的理解が有用な場合

## When NOT to use

- データが存在しない場合（推測で図を描かない）
- 複雑な図形・UML図・アーキテクチャ図（別ツール推奨）
- 画像生成が必要な場合（PNG/SVG生成は対象外）

---

## Inputs

### 必須
- なし（コンテキストから自動判定）

### 任意

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| `type` | 自動判定 | 可視化タイプを明示指定（bar/trend/heatmap/tree/table/status/flow） |
| `output` | ターミナル | 出力先（terminal/file/daily-note） |
| `width` | 20 | グラフ幅（文字数） |
| `target` | コンテキスト | 可視化対象データを明示指定 |

## 実行フロー

### 1. コンテキスト判定

**直前の会話・ファイル・データから「何を可視化すべきか」を推定**:

1. 明示的な対象指定があればそれに従う
   - 例: 「このディレクトリ構造を可視化して」→ ツリー表示
   - 例: 「Git履歴を可視化して」→ コミットタイムライン

2. 会話に数値データが含まれる場合
   - 比率・割合 → 横棒グラフ
   - 時系列データ → トレンドチャート
   - カテゴリ別集計 → 棒グラフ + 表

3. データ種別が不明な場合のみ、1回だけユーザーに確認
   - 「何を可視化しますか？ (例: ディレクトリ構造、数値データ、タスク進捗)」

**絶対厳守**: 推測でデータを補完しない。存在するデータのみを使用。

---

### 2. データ種別 → 可視化手法マッピング

| データ種別 | 可視化手法 | 適用例 | 出力例 |
|-----------|-----------|-------|-------|
| 比率・割合 | 横棒グラフ | カテゴリ別支出、タスク進捗 | `食費 █████░░░ 65%` |
| 時系列推移 | トレンドチャート | 週次・月次データ推移 | `Week 1: ████░░ 40% ↗️` |
| 完了/未完了 | ヒートマップ | 習慣トラッキング、カレンダー | `✅✅⬜✅✅⬜✅` |
| 構造・階層 | ツリー | ディレクトリ、組織図 | `├── dir1/` |
| 比較（2者以上） | 並列テーブル | 前月比、複数プロジェクト比較 | Markdown表 + 差分色 |
| 分布・集計 | 棒グラフ+表 | 頻度分析、ランキング | `████ 45件` |
| 状態・段階 | ステータス | 段階的進捗、評価 | `🟥🟧🟨🟩 (0-100%)` |
| 依存・フロー | 矢印図 | ワークフロー、依存関係 | `A → B → C` |

**判定ロジック**:
```
1. データに時刻・日付フィールドがある → 時系列推移
2. データに階層構造がある → ツリー
3. データが完了/未完了の2値 → ヒートマップ
4. データに比率・割合が含まれる → 横棒グラフ
5. データが比較対象を含む → 並列テーブル
6. データに評価・段階がある → ステータス
7. データに接続・依存関係がある → 矢印図
8. いずれにも該当しない → 棒グラフ+表
```

### 3. 可視化手法の出力ルール

#### 3-1. 横棒グラフ（比率・割合）

**基本ルール**:
- 幅: 20文字（調整可）
- バー: `█` (filled) / `░` (empty)
- ラベル + バー + 数値 + 割合

**フォーマット**:
```
カテゴリ名  ████████████░░░░░░░░  65% (¥45,000)
```

**生成例**:
```
カテゴリ別支出（2026-01）:

食費       ████████████░░░░░░░░  65% (¥45,000)
住居       ██████████████████░░  90% (¥80,000)
交通       ████░░░░░░░░░░░░░░░░  20% (¥15,000)
娯楽       ████████░░░░░░░░░░░░  40% (¥28,000)
```

**詳細**: `references/patterns.md` の「横棒グラフテンプレート」参照

---

#### 3-2. トレンドチャート（時系列推移）

**基本ルール**:
- 週次/月次/日次の推移を表示
- トレンド矢印: ↗️ (上昇) / → (横ばい) / ↘️ (下降)
- 棒グラフ + 数値 + 変化率

**フォーマット**:
```
Week N: ████████████░░░░░░░░ XX% (+YY%)
```

**生成例**:
```
Exercise - 週次推移（8週間）:

Week 1: ████░░░░░░░░░░░░░░░░ 20%
Week 2: ████████░░░░░░░░░░░░ 40% ↗️
Week 3: ████████████░░░░░░░░ 60% ↗️
Week 4: ████████████████░░░░ 80% ↗️
Week 5: ████████████████░░░░ 80% →
Week 6: ██████████████░░░░░░ 70% ↘️
Week 7: ████████████████░░░░ 80% ↗️
Week 8: ████████████████████ 100% 🎉

トレンド: 改善傾向 📈 (+80% from Week 1)
```

**詳細**: `references/patterns.md` の「トレンドチャートテンプレート」参照

---

#### 3-3. ヒートマップ（完了/未完了）

**基本ルール**:
- 絵文字: ✅ (完了) / ⬜ (未完了) / ⚫ (未記録)
- 週次カレンダー形式（7日1行）
- 凡例必須

**フォーマット**:
```
Week N: ✅✅✅✅✅⬜⬜
```

**生成例**:
```
Exercise - 直近30日間:

Week 1: ✅✅✅✅✅⬜⬜
Week 2: ✅✅✅⬜✅✅✅
Week 3: ✅✅✅✅✅✅⬜
Week 4: ✅✅✅✅✅✅✅

凡例: ✅ 完了 | ⬜ 未完了 | ⚫ 未記録
完了率: 23/28 (82%)
現在のストリーク: 7日 🔥
```

**詳細**: `references/patterns.md` の「ヒートマップテンプレート」参照

---

#### 3-4. ツリー（構造・階層）

**基本ルール**:
- 記号: `├──` (中間) / `└──` (最後) / `│` (継続)
- 深さ制限: デフォルト3階層（調整可）
- ファイル数が多い場合は省略表示

**フォーマット**:
```
root/
├── dir1/
│   ├── file1.txt
│   └── file2.txt
└── dir2/
    └── file3.txt
```

**生成例**:
```
プロジェクト構造:

project/
├── src/
│   ├── components/
│   │   ├── Header.tsx
│   │   └── Footer.tsx
│   └── pages/
│       └── index.tsx
├── public/
│   └── images/
└── README.md
```

**詳細**: `references/patterns.md` の「ツリーテンプレート」参照

---

#### 3-5. 並列テーブル（比較）

**基本ルール**:
- Markdown表形式
- 差分を色付き絵文字で表示: 🟢 (増加) / 🔴 (減少) / ⚪ (変化なし)
- 前期比・前月比・前年比などを明示

**フォーマット**:
```
| 項目 | 今月 | 前月 | 前月比 |
|------|------|------|--------|
| 収入 | ¥XXX | ¥XXX | +X% 🟢 |
```

**生成例**:
```
月次収支比較:

| 項目 | 今月 | 前月 | 前月比 |
|------|------|------|--------|
| 収入合計 | ¥450,000 | ¥420,000 | +7.1% 🟢 |
| 支出合計 | ¥280,000 | ¥295,000 | -5.1% 🟢 |
| 収支差額 | ¥170,000 | ¥125,000 | +36.0% 🟢 |
| 貯蓄率 | 37.8% | 29.8% | +8.0pt 🟢 |
```

**詳細**: `references/patterns.md` の「比較テーブルテンプレート」参照

---

#### 3-6. 棒グラフ+表（分布・集計）

**基本ルール**:
- ソート済み（降順）
- 合計行を追加
- バー + 数値 + 割合

**生成例**:
```
コミット数（メンバー別）:

Alice  ████████████████████ 120件 (40%)
Bob    ██████████████░░░░░░  85件 (28%)
Carol  ██████████░░░░░░░░░░  60件 (20%)
Dave   ██████░░░░░░░░░░░░░░  35件 (12%)
──────────────────────────────────────
合計                       300件 (100%)
```

**詳細**: `references/patterns.md` の「集計テンプレート」参照

---

#### 3-7. ステータス（状態・段階）

**基本ルール**:
- 段階色: 🟥 (0-25%) / 🟧 (26-50%) / 🟨 (51-75%) / 🟩 (76-100%)
- 閾値を明示

**生成例**:
```
プロジェクト進捗:

設計フェーズ   🟩🟩🟩🟩 100% (完了)
実装フェーズ   🟨🟨🟨░ 75% (進行中)
テストフェーズ 🟧░░░ 25% (開始)
デプロイ       🟥░░░  0% (未着手)

色凡例: 🟥 0-25% | 🟧 26-50% | 🟨 51-75% | 🟩 76-100%
```

**詳細**: `references/patterns.md` の「ステータステンプレート」参照

---

#### 3-8. 矢印図（依存・フロー）

**基本ルール**:
- 矢印: `→` (順次) / `⇒` (強調) / `↓` (分岐)
- 分岐・合流を表現
- 簡潔に（複雑なフローは別ツール推奨）

**生成例**:
```
デプロイフロー:

開発 → テスト → ステージング → 本番
         ↓
      バグ修正 ⇒ 再テスト
```

**詳細**: `references/patterns.md` の「フローテンプレート」参照

### 4. 出力先

| 出力先 | 動作 |
|-------|------|
| ターミナル（デフォルト） | チャットに直接表示 |
| ファイル | 指定パスに書き出し（`output=file:path/to/file.md`） |

**例**:

## エラーハンドリング

| パターン | 検出方法 | 対処 |
|---------|---------|------|
| データなし | コンテキスト確認 | 「可視化対象が見つかりません。データを提示してください」 |
| データ種別不明 | 判定ロジック不一致 | ユーザーに確認（1回のみ） |
| 出力先ファイルなし | ファイル存在チェック | ファイル作成 or エラー表示 |
| Daily Noteなし | 今日のDaily Note確認 | 「Daily Noteが存在しません」 |

## 絶対厳守ルール

### データ取得

1. **推測でデータを補完しない**: 存在するデータのみを使用
2. **欠損は明示的に報告**: 存在しないデータは欠損として報告
3. **判定ロジックに従う**: データ種別の判定は明確な基準に基づく

### 可視化手法

1. **詳細が必要な場合は `references/patterns.md` を参照**
2. **バー幅は20文字を基本**（調整可）
3. **凡例は必ず表示**（ヒートマップ、ステータス等）
4. **データが複雑な場合は簡略化**（階層は3階層まで、TOP5まで等）

### 出力

1. **デフォルトはターミナル表示**
2. **ファイル書き込み前に存在確認**
3. **Daily Note追記前に存在確認**

## DoD (Definition of Done)

### 必須条件

1. 「可視化して」等のトリガーワードで発動
2. コンテキストから可視化対象を自動判定
3. データ種別に応じた8種類の可視化手法を実装
4. 横棒グラフ（比率・割合）
5. トレンドチャート（時系列推移）
6. ヒートマップ（完了/未完了）
7. ツリー（構造・階層）
8. 並列テーブル（比較）
9. 棒グラフ+表（分布・集計）
10. ステータス（状態・段階）
11. 矢印図（依存・フロー）
12. デフォルトはターミナル表示
13. ファイル出力に対応
14. Daily Note追記に対応
15. 詳細テンプレートは `references/patterns.md` に分離
16. データがない場合は適切なエラーメッセージ
17. 判定不明時は1回のみユーザー確認
18. 推測でデータを補完しない

### 品質条件

19. 他スキル（exec-monthly-finance、exec-weekly-review、habit-tracker-integrator）と同じSKILL.md構造
20. 既存の可視化パターンを踏襲
21. バー幅20文字を基本、調整可能
22. 凡例・トレンド矢印・色付き絵文字を適切に使用
